# Zstandard标准压缩

:::danger 需要协助
本节内容亟待完善。如果您能提供帮助，请参阅[贡献指南](../contribution-guide.mdx)开始参与贡献！

Zstandard是Facebook开发的一种压缩算法，以其极快的解压速度著称。该算法于2015年初发布，被广泛应用于多种场景。其设计初衷是在保持与旧版基于Deflate的压缩算法（如[ZIP](../data/zip.mdx)或[gzip](../data/gzip.mdx)）相似压缩率的同时，实现更快的整体性能。实际测试中，其压缩效果接近纯LZMA算法（[XZ](../data/xz.mdx)和[7-zip](../data/7z.mdx)的组成部分），但速度明显更快。

虽然`.tar.zstd`归档文件不如`.tar.xz`或`.tar.gz`普及，但Zstandard已是开源软件领域广受欢迎的压缩工具。它已被集成到FreeBSD内核和Linux内核中，并作为btrfs、squashfs、bcachefs和OpenZFS文件系统的透明压缩方法。文件系统压缩是指始终对存储文件进行实时压缩的方案，可显著减少存储空间占用。

命令行工具`zstd`默认支持1至19级压缩，使用`--ultra`参数时可扩展至22级。Arch Linux所有软件包均采用zstd 20级压缩，相比XZ格式，解压速度快14倍，但平均文件体积仅增加0.8%。该算法在游戏模拟器领域也很受欢迎，许多模拟器支持zstd压缩游戏文件。自2020年发布的6.3.8版起，ZIP文件格式标准已支持编号93的Zstandard压缩。Chromium 118版本起通过实验性标志支持zstd内容编码，未来可能与[Brotli](./brotli.mdx)竞争网页压缩标准。苹果的LZFSE算法据称与Zstandard 6级压缩效果相似。

Zstandard有望在现代应用场景中与各类主流压缩算法展开竞争。如果作为内容分发格式得到普及，其超快速、低资源占用的解码特性可能取代Brotli——尽管需要牺牲少量压缩率。在更高压缩级别下，其归档体积往往优于Brotli。未来，`.tar.zst`可能取代7-zip、ZIP等归档格式，在不同性能设备上实现高速解码。

## 使用方法
:::note
本指南针对`zstd`命令行工具编写，但peazip、7zip等图形化归档工具已逐步增加对zstd的支持。
:::

## 压缩文件
与其他压缩工具类似，如需压缩多个文件，建议先使用[tar](./tar.mdx)打包：
```bash
zstd -# {文件} -o {文件}.zstd
```
`-#`为压缩等级参数，例如`-3`、`-6`。默认支持1-19级，配合`-ultra`参数可扩展至22级。

## 解压文件
```bash
zstd -d {文件}.zstd -o 输出文件
```
