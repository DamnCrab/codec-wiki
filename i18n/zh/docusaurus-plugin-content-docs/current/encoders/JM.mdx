# JM 编码器

:::danger 招募协助
本节内容亟待完善。如果您能提供帮助，请查看我们的[贡献指南](../contribution-guide.mdx)开始参与贡献！
:::

JM是最初的[H.264](../video/AVC.mdx)参考编码器实现，早于[x264](../encoders/x264.mdx)等替代方案。由于x264在编码效率和视觉感知优化方面表现出色，JM与其他MPEG参考编码器（如[HM](../encoders/HM.mdx)和[VTM](../encoders/VTM.mdx)）一样，因使用复杂度高和编码速度慢而成为极少使用的特殊方案。通常x264的编码效率优于JM。
