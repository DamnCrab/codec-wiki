# SVT-AV1-PSY

:::info 社区分支版本
本文介绍的是SVT-AV1的一个分支版本SVT-AV1-PSY。如需了解主线的SVT-AV1编码器，请参阅我们的[SVT-AV1百科条目](./SVT-AV1.mdx)。
:::

SVT-AV1-PSY是一个旨在增强AV1可扩展视频编码技术的项目，通过感知优化实现心理视觉最优的AV1编码。其终极目标是打造AV1编码器中感知质量最佳的实现方案。该项目由专注的开发者和贡献者团队共同推进，致力于提升编码器的[心理视觉质量](../introduction/psychovisual.mdx)。

该项目由[Gianni Rosato](https://github.com/gianni-rosato)、[Julio Barba](https://github.com/juliobbv)和[Clybius](https://github.com/Clybius)维护，社区贡献者包括[SVT-AV1-PSY AUR包](https://aur.archlinux.org/packages/svt-av1-psy-git)维护者[BlueSwordM](https://github.com/BlueSwordM)以及[aom-av1-psy](https://github.com/BlueSwordM/aom-av1-psy)的原作者。

开发过程包含社区测试与优化，确保编解码器达到最佳性能。团队使用包括主观分析在内的多种工具和方法来改进编解码器性能，其中[SSIMULACRA2](../metrics/SSIMULACRA2.mdx)和[XPSNR](../metrics/XPSNR.mdx)被广泛用作质量评估指标。但项目明确目标并非提升指标分数，而是提高编码器的整体感知质量——因此常会出现为追求心理视觉保真度而降低指标表现的情况。

更多项目信息可访问[项目官网](https://svt-av1-psy.com/)。该仓库已于2025年4月30日归档，作者在[声明](https://github.com/psy-ex/svt-av1-psy/blob/master/README.md)中正式宣布项目终止，并提及由BlueSwordM维护的分支版本[svt-av1-psyex](https://github.com/BlueSwordM/svt-av1-psyex)。

SVT-AV1-PSY贡献者与开放媒体联盟(Alliance for Open Media)及未参与本项目的上游SVT-AV1贡献者无任何关联。

## 微版本框架

SVT-AV1-PSY包含许多主线版本未纳入的重要特性，因此采用独特的微版本发布机制。带字母后缀的版本（如`v2.0.0-A`）表示包含PSY专属新特性，无后缀版本则与主线版本同步发布。

## 新增特性

SVT-AV1-PSY包含多项主线版本不具备的功能，旨在提升AV1编码的视觉质量并为不同编码场景提供更灵活的配置选项。完整变更列表详见[SVT-AV1-PSY说明文件](https://github.com/gianni-rosato/svt-av1-psy/blob/master/README.md)。

## 安装指南

<Tabs>
    <TabItem value="unixlike" label="Linux & macOS">
        *x86_64架构Linux用户可通过[rAV1ator CLI](../utilities/rav1ator-cli.mdx)获取预编译的AVX2优化版本，但建议始终从源码构建*

        克隆仓库并进入构建目录：
        ```bash title="克隆SVT-AV1-PSY"
        git clone https://github.com/gianni-rosato/svt-av1-psy
        cd svt-av1-psy/Build/linux
        ```

        执行构建命令（需CMake 3.16+及GCC/Clang）：
        ```bash title="构建发行版"
        ./build.sh release
        ```
        ```bash title="使用Clang静态构建编码器（启用LTO）"
        ./build.sh jobs=8 all cc=clang cxx=clang++ no-dec enable-lto static native
        ```

        编译产物位于`Bin/Release`目录。若只需编码器，添加`no-dec`参数可跳过解码器编译。

        性能调优提示：可通过PGO（性能导向优化）提升编码效率。以下脚本假设存在/dev/shm目录下的.y4m测试文件：
        ```bash title="PGO优化构建"
        git clone https://gitlab.com/AOMediaCodec/SVT-AV1/
        cd SVT-AV1/Build/linux
        ./build.sh cc=gcc cxx=g++ enable-lto enable-pgo static native jobs=$(nproc) pgo-dir=/dev/shm pgo-videos=/dev/shm release
        ```
    </TabItem>
    <TabItem value="windows" label="Windows">
        **MSYS2**是Windows平台推荐构建环境：
        1. 安装依赖：
        ```bash
        pacman -Syu --needed git mingw-w64-ucrt-x86_64-toolchain mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja mingw-w64-ucrt-x86_64-yasm
        ```
        2. 可选Clang安装：
        ```bash
        pacman -Syu --needed mingw-w64-ucrt-x86_64-clang
        ```
        后续步骤与Linux/macOS相同，注意CMake需添加`-G "Ninja"`参数。
    </TabItem>
</Tabs>
