# 前言

多媒体压缩技术彻底革新了我们在网络及其他领域的通信方式。它让众多突破性平台得以实现原本不可能的丰富体验，使我们能够以创新的方式传递信息、表达情感并建立人际连接。这项技术是现代互联网的无名英雄。然而，要获取关于编解码器技术的准确、专业且经过实践验证的知识却非易事。本维基旨在揭开多媒体压缩领域的神秘面纱，同时汇聚编解码器爱好者共同构建知识宝库。

### 非定位说明

本编解码器维基**并非理解数学原理、研究论文、专利政治或具体编解码器实现规范的高度精确来源**。诸如维基百科等平台已对这些细节进行了精准可靠的阐述。我们关注的重点在于让高层次信息——特别是与压缩工具使用及应用相关的知识——更易于获取；现阶段我们聚焦实践应用而非理论原理。了解编解码器的理论工作机制与掌握其最佳使用场景及配套工具应用是截然不同的。

### 何为编解码器

编解码器（coder/decoder缩写）是依据特定规范处理数字媒体或数据的系统，通常指对数字媒体进行压缩与解压缩的工具。编解码器用于将媒体编码以便存储传输，随后解码以供播放编辑等用途。多媒体编解码器通过两种方式压缩：[丢弃次要数据](../introduction/psychovisual.mdx)的[有损](../introduction/video-artifacts.mdx)压缩可减小文件体积；或采用无损压缩技巧，在保持与输入媒体数学一致性同时缩减文件体积。无损压缩可完全还原原始数据，而有损压缩因丢弃数据无法实现这点。常见编解码器应用包括：

- 视频压缩：如[H.264](../video/AVC.mdx)、[VP9](../video/VP9.mdx)和[AV1](../video/AV1.mdx)等视频编解码器可将数字视频大幅压缩，便于流媒体传输与存储。视频编解码器可在录制时或分发前编码视频流，播放时再解码，显著提升分享效率并节省存储与带宽。
- 音频压缩：如[MP3](../audio/MP3.mdx)、[AAC](../audio/AAC.mdx)和[Opus](../audio/Opus.mdx)等音频编解码器可压缩音乐播客等音频文件，便于分发存储。
- 图像压缩：无论是经典的[JPEG](../images/JPEG.mdx)还是新兴的[JPEG-XL](../images/JXL.mdx)，图像编解码器的核心目标都是高效压缩同时保持多功能特性。色深、HDR、透明度、色彩空间信息、EXIF数据等多元因素使图像压缩成为复杂课题。
- 数据压缩：如[ZIP](../data/zip.mdx)和[zstd](../data/zstd.mdx)等通用压缩算法可处理任意数据类型（网页资源、可执行文件、文本归档甚至完整文件系统），不仅限于多媒体数据。

简言之，**编解码器通过复杂算法实现媒体编码解码，优化存储与传输效率**。它们是数字媒体录制、压缩、传输及回放的核心技术。不同编解码器根据应用场景在压缩效率、质量、算力需求、兼容性及功能特性间寻求平衡。

### 学习准备

以下为非正式需求清单：

您将显著受益于：
- 耐心
- 学习意愿、好奇心及遵循指导的自觉性
- 基础至中级的计算机操作能力

若仅关注工具学习，建议具备：
- 高性能CPU硬件，可缩短大型编码任务等待时间
- 命令行工具使用基础，或零基础但具备学习动力
- 运行Arch系Linux发行版的设备（Manjaro除外）

:::info 为何选择Arch？
多数编码工具均可通过包管理器直接获取，且该前沿发行版能确保工具持续更新。对于滤镜处理，所有Vapoursynth插件均已收录于Arch用户仓库(AUR)，配合[yay](https://github.com/Jguer/yay)等AUR助手可轻松实现安装与版本控制。
:::
