# 矩阵系数

矩阵系数表示从YUV转换到RGB时使用的乘法矩阵。与基色类似，这些整数值在通用规范中定义，因此所有编码和播放工具中的值都相同。

可用的矩阵系数值如下：

### 0: 恒等矩阵

指定使用恒等矩阵，即该数据已处于RGB兼容的色彩空间中。

该矩阵系数设置用于以下标准：

- GBR（通常称为RGB）
- YZX（通常称为XYZ）
- IEC 61966-2-1 sRGB
- SMPTE ST 428-1 (2019)

### 1: BT.709

BT.709是现代高清视频使用的标准，是最安全的默认假设。

该矩阵系数设置用于以下标准：

- Rec. ITU-R BT.709-6
- Rec. ITU-R BT.1361-0 常规色域系统和扩展色域系统（历史）
- IEC 61966-2-4 xvYCC709
- SMPTE RP 177 (1993) Annex B

### 2: 未指定

此值表示视频未设置色彩矩阵，播放器需自行决定使用哪个值。

mpv在此情况下会使用以下启发式规则：

```
if width >= 1280 || height > 576 {
    "BT.709"
} else {
    "SMPTE 170M"
}
```

### 4: BT.470M

BT.470M是美国模拟电视系统使用的标准。

### 5: BT.470BG

BT.470BG是欧洲(PAL)电视系统和DVD使用的标准。

该矩阵系数设置用于以下标准：

- Rec. ITU-R BT.470-6 System B, G（历史）
- Rec. ITU-R BT.601-7 625
- Rec. ITU-R BT.1358-0 625（历史）
- Rec. ITU-R BT.1700-0 625 PAL和625 SECAM
- IEC 61966-2-1 sYCC
- IEC 61966-2-4 xvYCC601

### 6: SMPTE 170M

SMPTE 170M是NTSC电视系统和DVD使用的标准，其矩阵系数与BT.470BG相同。

该矩阵系数设置用于以下标准：

- Rec. ITU-R BT.601-7 525
- Rec. ITU-R BT.1358-1 525或625（历史）
- Rec. ITU-R BT.1700-0 NTSC
- SMPTE ST 170 (2004)

### 7: SMPTE 240M

SMPTE 240M是HDTV早期（1988-1998年）使用的过渡标准。

### 8: YCgCo

YCoCg色彩模型（又称YCgCo色彩模型）是通过简单变换RGB色彩空间形成的色彩空间，包含一个亮度值和两个色度值（色度绿和色度橙）。

### 9: BT.2020非恒定亮度

BT.2020是超高清视频（4K及以上）使用的标准，可搭配HDR使用（HDR由传递特性定义）。如果不确定选择非恒定亮度还是恒定亮度版本，通常应选择非恒定亮度。

如果视频未设置矩阵系数，更安全的做法是默认假设为BT.709。BT.2020视频应已设置其色彩元数据。

该矩阵系数设置用于以下标准：

- Rec. ITU-R BT.2020-2（非恒定亮度）
- Rec. ITU-R BT.2100-2 Y′CbCr

### 10: BT.2020恒定亮度

这是BT.2020的恒定亮度变体，使用YcCbcCrc色彩空间表示。除非明确需要，否则建议选择非恒定亮度版本。

### 11: SMPTE 2085

SMPTE 2085是与XYZ色彩空间中HDR信号搭配使用的标准，实际应用中极为罕见。

### 12: 色度推导非恒定亮度

不确定其具体应用场景。

### 13: 色度推导恒定亮度

不确定其具体应用场景。

### 14: ICtCp

ICtCp是由杜比公司开发的替代色彩空间，用于HDR和广色域视频（杜比总是喜欢搞这类额外技术），实际应用中极为罕见。
