# 色彩原色

本节详述视频编码中保持色彩准确性的三大核心设置之一，这三项设置分别为：原色（primaries）、色彩矩阵（color matrix）和传递特性（transfer characteristics）。

色彩原色用于标定红、蓝、绿三原色的精确坐标。[众多标准并存的现象](https://xkcd.com/927/)有其历史原因，本指南不深入探讨历史背景，而将重点说明各类原色标准的适用场景。

注意：您可以使用[MediaInfo](https://mediaarea.net/en/MediaInfo)等工具查看视频中设置的原色参数。若未设置参数值，播放器将自行推断。现代视频通常默认采用BT.709标准，但具体仍需根据视频源和分辨率调整。强烈建议编码时正确设置这些参数。

每个设置至少对应一个名称和一个整数值——多数编码软件接受名称标识，而Vapoursynth和MKVToolnix等工具则采用整数值。这些整数值遵循国际规范，在所有编码和播放工具中保持一致。

### 1: BT.709

BT.709是现代高清视频的通用标准，可作为安全默认值。

该原色标准应用于以下规范：
- ITU-R BT.709-6建议书
- ITU-R BT.1361-0常规色域系统及扩展色域系统（历史）
- IEC 61966-2-1 sRGB或sYCC
- IEC 61966-2-4
- 电影电视工程师协会（SMPTE）RP 177（1993）附录B

### 2: 未指定

表示视频未设置原色参数，由播放器自行决定。

mpv播放器在此情况下的判定逻辑：
```
if matrix == "BT.2020" {
    "BT.2020"
} else if matrix == "BT.709" {
    "BT.709"
} else if width >= 1280 || height > 576 {
    "BT.709"
} else if height == 576 {
    "BT.470BG"
} else if height == 480 || height == 488 {
    "SMPTE 170M"
} else {
    "BT.709"
}
```

### 4: BT.470M

美国模拟电视系统历史标准。

应用于：
- ITU-R BT.470-6系统M（历史）
- 美国国家电视系统委员会1953年彩色电视传输标准建议
- 美国联邦通信委员会（2003）47 CFR 73.682(a)(20)

### 5: BT.470BG

欧洲PAL电视系统及DVD采用的标准。

应用于：
- ITU-R BT.470-6系统B/G（历史）
- ITU-R BT.601-7 625线
- ITU-R BT.1358-0 625线（历史）
- ITU-R BT.1700-0 625线PAL及SECAM

### 6: SMPTE 170M

NTSC电视系统及DVD采用的标准。
- ITU-R BT.601-7 525线
- ITU-R BT.1358-1 525/625线（历史）
- ITU-R BT.1700-0 NTSC
- SMPTE ST 170（2004）

### 7: SMPTE 240M

高清电视发展初期（1988-1998）的过渡标准，其原色与SMPTE 170M等效。

### 8: 电影胶片

使用C光源的通用胶片标准。

### 9: BT.2020

超高清视频（4K及以上）标准，可配合HDR使用（HDR由传递特性定义）。

未设置原色的视频建议默认采用BT.709，实际BT.2020视频应已包含色彩元数据。

应用于：
- ITU-R BT.2020-2
- ITU-R BT.2100-2

### 10: SMPTE 428

数字影院发行母版（DCDM）专用标准。

应用于：
- SMPTE ST 428-1（2019）
- （ISO 11664-1规定的CIE 1931 XYZ色彩空间）

### 11: DCI-P3

由[数字影院倡导组织](https://www.dcimovies.com)制定的色域标准，其色度端点与AdobeRGB相似（后者广泛应用于平面艺术和摄影领域），色彩范围显著大于sRGB。

### 12: Display P3

苹果公司自2016年起在iPhone、iPad、iMac、MacBook及外接显示器采用的标准。

与DCI-P3的区别：
- 白点采用与sRGB/AdobeRGB相同的6500K（DCI-P3不同）
- 伽马值采用近似2.2的sRGB标准（DCI-P3为2.6）

### 22: EBU Tech 3213

1975年制定的PAL/SECAM电视系统色彩空间标准，定义了原色色度与白点参考。现已被SMPTE 170M等新标准取代，其传递函数从未明确定义。
