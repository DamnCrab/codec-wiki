# QSV（快速同步视频技术）

:::caution 待审核
本条目内容可能不完全准确，正在等待进一步审核以评估信息质量。
:::

QSV（QuickSync Video）是英特尔集成在其现代带核显CPU及Arc显卡中的硬件视频编解码平台。该技术允许应用程序将视频编码、解码和处理任务卸载至英特尔专用多媒体硬件上的媒体引擎，相比基于CPU的软件编码通常能带来显著的性能提升。QSV与英伟达的[NVENC](./nvenc.mdx)和AMD的[AMF](./amf.mdx)形成竞争（自苹果转向自研芯片后，英特尔Mac上的QSV还需与macOS设备上的苹果[VideoToolBox](./videotoolbox.mdx)竞争）。

QSV的核心目标是通过利用英特尔图形处理器中的专用固定功能硬件，加速视频编码、解码和处理工作负载。这些专用硬件独立于通用计算单元，专为多媒体任务设计。QSV旨在实现高效能编解码的同时保持出色能效表现。

QSV支持广泛的视频编解码器，包括最新独立/集成显卡上的[H.264](../video/AVC.mdx)、[H.265](../video/HEVC.mdx)、[VP9](../video/VP9.mdx)以及[AV1](../video/AV1.mdx)。其最大优势在于广泛集成于现代英特尔核显CPU中，使得硬件加速视频编码能在多种系统上实现。应用程序可通过英特尔Media SDK、VA-API等接口，或通过集成[FFmpeg](../utilities/ffmpeg.mdx)、GStreamer等流行多媒体框架轻松调用QSV加速。

采用QSV的硬件加速视频编码通常在压缩效率略低于软件编码方案的情况下，显著提升编码性能。当进行超实时编码时（如60-200fps），能以更低比特率生成更高质量输出。这对需要实时编码的应用场景（如直播、视频会议和游戏录制）特别有利。

虽然QSV并非为高压缩率的离线文件编码（质量优先于速度的场景）而设计，但值得注意的是：在压缩效率方面，QSV几乎总是优于AMD的AMF，并与英伟达的NVENC保持竞争力。
