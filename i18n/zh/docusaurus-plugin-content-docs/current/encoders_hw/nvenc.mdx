# NVENC

:::caution 内容待审核

:::

NVENC是NVIDIA显卡中集成的专用硬件视频编码引擎。该技术允许应用程序调用专用的多媒体编码硬件来加速视频编码任务，相比基于CPU的软件编码能显著提升性能。它与英特尔的[QSV](./qsv.mdx)和AMD的[AMF](./amf.mdx)等同类框架形成竞争关系。

NVENC的主要目的是将计算密集型视频编码工作负载从CPU转移到GPU的专用多媒体硬件上，从而释放CPU资源处理其他任务。这一特性在需要快速视频编码的场景中尤为有利，例如屏幕录制、直播和视频会议。

NVENC支持多种主流视频编解码器，包括[H.264](../video/AVC.mdx)、[H.265](../video/HEVC.mdx)、[VP9](../video/VP9.mdx)，以及最新显卡上支持的[AV1](../video/AV1.mdx)。它为这些编解码器提供硬件加速编码能力，通常能实现实时或超实时的编码性能，具体表现取决于分辨率、比特率和硬件性能。

虽然NVENC在编码速度上表现卓越，但与采用较慢预设模式的现代高质量CPU软件编码器相比，其压缩效率通常会有所牺牲。

NVENC设计初衷是便于集成到各类多媒体框架和应用中。主流工具如[FFmpeg](../utilities/ffmpeg.mdx)、OBS Studio等都支持该技术，开发者无需进行底层编程即可无缝调用GPU加速编码。

相较于AMD的AMF和英特尔的QSV，NVENC以高编码性能、低延迟以及与NVIDIA显卡的广泛兼容性著称。它在游戏主播、内容创作者和需要快速编码的视频专业人员中尤其受欢迎。就视频压缩效率而言，NVENC与QSV各有胜负，而AMF通常稍逊一筹。
