# AMF 加速媒体框架

:::caution 待审阅
本条目内容可能不完全准确，正在等待进一步审查以评估信息质量。
:::

AMD GPU 的 AMF 技术允许应用程序利用显卡内置的专用视频编解码硬件。

AMD 高级媒体框架（AMF）是由 AMD 开发的底层 API，使开发者能够在 AMD GPU 上实现硬件加速的视频编码与解码。通过调用 GPU 媒体处理单元中的专用硬件，视频编解码任务可从 CPU 卸载，从而显著提升处理速度和效率。AMF 为应用程序提供多媒体处理功能，与英伟达的 [NVENC](./nvenc.mdx) 和英特尔的 [QSV](./qsv.mdx) 提供同类技术竞争。

AMF 支持多种视频编解码器，包括 [H.264](../video/AVC.mdx)、[H.265](../video/HEVC.mdx)、[VP9](../video/VP9.mdx)，以及最新支持显卡上的 [AV1](../video/AV1.mdx)。GPU 编码能力特别适合实时视频内容压缩场景，此时处理速度比编码效率更为重要。

使用 AMF 的硬件加速视频编码通常能以较低的压缩效率显著提升编码性能。在超实时编码场景（如60-200 fps）下，通常能在较低比特率下获得更高质量输出。这对需要实时编码的应用（如直播、视频会议和游戏录制）尤为有利。

但相较硬件编码，软件编码方案几乎总能实现更高的比特效率。对于离线转码和存储场景，通常更推荐使用软件编码。特别是 AMF 的压缩效率并不突出，这受限于 AMD 相对较弱的媒体处理单元——其性能通常逊于英伟达、英特尔和苹果的硬件实现方案。

AMF 设计时考虑了与主流媒体框架的无缝集成，如 [FFmpeg](../utilities/ffmpeg.mdx) 和 GStreamer。这些框架通常内置 AMF 支持，开发者无需进行底层 API 编程即可轻松实现硬件加速编码。

AMF 兼容多种 AMD GPU，包括独立显卡和集成显卡解决方案。支持 Windows 和 Linux 等多种操作系统，使不同平台的开发者都能使用该技术。
